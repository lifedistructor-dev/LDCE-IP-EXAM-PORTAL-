CH 14Structuring Database for Accounting — ER Model

Very Important
Entity: Anything in the real world with independent existence — physical (car, person) or conceptual (company, account, voucher).
Attributes: Properties describing an entity (e.g., Code, Name for Accounts).
🏷 5 Types of Attributes
1. Composite vs Simple — Name (composite: First/Middle/Last) vs Height (simple)
2. Single-valued vs Multi-valued — Height (single) vs Qualifications (multiple)
3. Stored vs Derived — Date of Birth (stored) vs Age (derived)
4. Null Values — attribute doesn't apply / unknown / doesn't exist
5. Complex Attributes — nested composite + multi-valued; () for composite, {} for multi-valued
Key/Identifier Attribute: An attribute with unique values that identifies each entity instance (e.g., Code for Accounts). Two+ attributes together = composite key. Entities with NO key attribute = Weak Entity Types (identified only via their owner entity, e.g., SupportDocuments depends on Vouchers).
🔗 Relationships — Key Concepts
Degree: Binary (2 entities) / Ternary (3 entities)
Role Names: role each entity plays (e.g., Employee = creator, Voucher = created)
Cardinality Ratio: 1:1, 1:N, N:1, N:M
Participation: Total (existence-dependent, double line in ER diagram) vs Partial (single line)
🎯 Probable Question: "Define Entity, Attribute, and Weak Entity Type with examples." / "Explain types of cardinality ratios." (5–6 marks each — very common)
Remember: Weak Entity = "No ID of its own — borrows its owner's."

🚨 Last-Day Revision Notes

  • Accounting PROCESS is conceptually identical in manual and computerised systems — only the mechanism changes.
  • Ready-to-use software = low cost + low secrecy; Tailored = high cost + high secrecy — inversely linked.
  • An entity type becomes "weak" only when it has NO key attribute of its own (needs an owner).
  • Cardinality ratio types: 1:1, 1:N, N:1, N:M — memorize with the Voucher:Employee (N:1) example.
  • Summary Query is the workhorse of accounting reports — always involves GROUP BY / aggregate functions.
Quick Trick — "TFQR": Tables store → Forms enter → Queries retrieve → Reports present.
