22.01 Key Definitions 🔴 VI▾
Postal Franking Machine: Stamping machine that impresses approved-design Dies on postal articles in payment of postage/fees.
Licensing Authority: Authorised to issue/renew/suspend/cancel the Franking Machine licence.
OEM (Original Equipment Manufacturer): The original manufacturer/technology provider of RMFS machines.
Franking Machine Vendor: OEM-authorised company/firm that sells/supplies/installs/maintains machines in a region (OEM can also be its own vendor).
Franking Machine User: Valid licence-holder who deposits franked mail.
CRN (Customer Reference Number): Unique number issued by Licensing Authority — same as the Licence Number.
Licence Identifier: Machine-specific unique number assigned by the OEM.

🧠 The whole RMFS process is system-driven — no manual posting is permitted anywhere in the cycle.

License fee is fee-only — cannot be used for franking/consumption. Deposited at POS Counter.

22.03 Recharge Process Flow 🟠 FAQ▾

Key rule: No partial upload permitted — the entire deposited recharge amount uploads to RMFS in one go. Cheque recharge subject to realisation.

User pays via Cash/Cheque/ECS-NEFT-RTGS to Postmaster's bank account
↓
Postal Assistant enters details in POS Franking screen (CRN/License ID)
↓
POS connects to SAP via BIZTALK → PA validates transaction
↓
SAP creates Fund File → transmitted to OEM
↓
OEM acknowledgement received → transaction complete; credit flows to GL 8446-00101200200
↓
User downloads funds from OEM server → starts franking
22.04 Booking Articles — Accountable vs Unaccountable 🟡 Concept▾

SOM (Statement of Mailing): Printed from Franking Machine; usage data flows Machine → OEM Server → SAP. Daily SOM entry (accountable + ordinary articles, separately) via POS back office — essential since it's linked to Rebate payment eligibility.

Interim credit flows to GL 120100101230100 — does not flow to CGA at this stage (no service rendered yet). Only after PA books the article against CRN does billing/payment document generate, utilising that credit.

Unaccountable articles: Handed over at POS/back office with SOM — SOM capture + daily server connection are prerequisites for revenue recognition.

22.05 Rebates & Refunds 🔴 VI▾
TypeRate / Condition
Base Rebate3% on franked value, min. consumption ₹5,000
Additional Rebate+2% for Pin-code-wise sorted franked mail presentation
Error RefundFace value less 5% deduction; claim within 1 month of impression date; impression must be legible

Rebate sanctioned by Sub-Postmaster/Postmaster of designated office; SOM capture via POS back office is mandatory for rebate payment. Rebates charged to GL 120100101230300; Refunds charged to GL 120100101230100.

All refund/cancellation/rebate processing is 100% system-driven via POS — no manual voucher posting under any circumstance.

Likely Q: What is the minimum franked-value consumption required to claim the 3% base rebate? → ₹5,000.

22.06 License Cancellation & Departmental Machines 🟠 FAQ▾

Cancellation: System-driven via POS Franking menu; debit posted to GL 8844600360 (Special GL Indicator "G").

Departmental Franking Machines: Use Notional Credit — a one-time activity, credit amount never altered thereafter. Used for both accountable & unaccountable articles. No Rebate allowed for departmental machine usage.

Settlement Process: Ensures recharges reconcile against consumption — credits under GL 8446 series only liquidate after the settlement accounts process runs.

⚡ One-Page Revision Sheet — Chapter XXII

🔴 License fee = ₹375, valid 5 years
🔴 Base Rebate = 3% (min. ₹5,000 consumption); Additional = +2% for pin-sorted mail
🔴 Refund = face value less 5%, claimed within 1 month
🔴 No partial recharge upload — full amount uploads in one go
🟠 Late renewal surcharge = ₹100
🟠 Renewal must be initiated ≥1 month before expiry
🟢 CRN = same as License Number
🟢 No Rebate for Departmental Franking Machine usage
🟢 Notional Credit = one-time, never altered afterward
🟢 Private machine spot-visit report = once a year
🟢 Entire RMFS cycle = 100% system-driven, zero manual posting
